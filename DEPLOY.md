# teambeam.blog — deploy and maintain (no coding needed)

You never run anything on your own computer. You add a text file on GitHub; Cloudflare rebuilds the whole site for you.

## One-time setup

1. Create a new GitHub repository and upload every file in this folder to it (keep the folder structure exactly — `build.js`, `content/`, `assets/`, `package.json` all at the top level).
2. In Cloudflare, go to **Workers & Pages → Create → Pages → Connect to Git** and pick that repository.
3. Set the build settings exactly:
   - **Framework preset:** None
   - **Build command:** `node build.js`
   - **Build output directory:** `site`
   - **Root directory:** `/` (leave as is)
4. Deploy. Cloudflare runs the build and publishes the `site` folder.
5. In **Custom domains**, add `teambeam.blog`.

That is the whole setup. It happens once.

## Adding a new article (about two minutes)

1. Open `ARTICLE-TEMPLATE.md` in this project and copy the block between START OF FILE and END OF FILE.
2. On GitHub, go into the `content` folder → **Add file → Create new file**.
3. Name it after the slug, ending in `.md` — for example `content/proving-an-offsite-worked.md`.
4. Paste the block, fill in every line, and **Commit**.
5. Cloudflare rebuilds automatically. Your article is live in a minute or two.

## What updates by itself

When you add one article file, the build regenerates all of this with no extra work:

- the article's own page
- the home page (Latest, and Start here)
- the correct pillar page
- the "Read more" and "More from this pillar" links on other articles
- the sitemap, the RSS and JSON feeds
- all structured data (JSON-LD) for search and AI answer engines

You never touch the home page or menus. They keep themselves in sync.

## Rules that keep the site consistent

- Every article needs `title`, `slug`, and a valid `pillar` code (P1–P10). If one is missing, the build stops and tells you which file — nothing broken ever goes live.
- Each pillar has one fixed colour. That is intentional: the look never drifts between updates.
- There is no comment section. Every article invites readers to write to start@teambeam.in.
- The "Talk to us" button sends India visitors to teambeam.in, US visitors to teambeam.us, and everyone else to teambeam.in/go — automatically, without ever making the reader feel sorted.

## Structured data (why search errors won't happen here)

All JSON-LD on this site is generated by the build from data, never typed by hand. The build also re-reads every page it writes and re-parses each JSON-LD block; if any block were invalid, the build fails and refuses to publish. The kind of "Unparsable structured data" error that a hand-typed stray quote causes cannot occur here.
